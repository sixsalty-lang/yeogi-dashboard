# 여기어때 국내여행 인사이트 대시보드 — 사용 설명서

개발자가 아니어도 따라 할 수 있도록 모든 단계를 적었습니다.
**키 없이도 바로 작동**하므로, 먼저 화면부터 확인하고 나중에 실데이터를 연결하세요.

---

## 0. 이 앱이 하는 일

| 탭 | 내용 | 데이터 출처 |
|----|------|------------|
| 📈 수요 예측 | 지역별 국내여행 월 방문자 추이 + 향후 예측 + **CSV 내려받기** | 한국관광 데이터랩 (KT 내국인) |
| 👥 연령대별 | 방문객의 연령대 분포 + 광고 집행 제안 | 한국관광 데이터랩 (내국인) |
| 📺 예능 여행지 | 예능에 나온 여행지 콘텐츠 자동 수집 (**프로그램명 자동 감지**) | 네이버 검색 API |

> **중요:** 방문자 데이터는 KT(내국인) / SKT(외국인)로 소스가 분리돼 있어,
> 이 앱은 **내국인 지표만** 사용합니다. 외국인이 섞인 데이터는 들어가지 않습니다.

### v2 개선 사항 (2026-07)
- **설날·추석 반영**: 국내여행 수요의 최대 변수인 명절 연휴가 예측 모델에
  실제로 작동하도록 수정했습니다. (음력이라 매년 바뀌는 양력 날짜를 2028년까지 내장)
- **CSV 내려받기**: 예측 수치를 버튼 한 번으로 내려받아 광고 보고서에 바로
  붙일 수 있습니다. 엑셀에서 한글이 깨지지 않게 처리돼 있습니다.
- **정직한 데이터 모드 표시**: 실데이터 연결 함수가 완성되기 전에는 키를
  입력해도 "🟡 샘플"로 표시됩니다. 샘플을 실데이터로 오해하는 일을 막기 위한 것으로,
  함수가 구현되면 자동으로 "🟢 실데이터"로 바뀝니다.
- **예능 프로그램명 자동 감지**: 수집된 콘텐츠에서 "1박 2일", "텐트 밖은 유럽" 등
  프로그램명을 찾아 카드에 라벨로 표시합니다. (`data_sources.py`의
  `KNOWN_SHOWS` 목록에 프로그램을 추가하면 감지 대상이 늘어납니다)
- 속도 개선(같은 조건 재조회 시 캐시 사용), 외부 텍스트 안전 처리 등.

## 1. 설치 (최초 1회)

### 1-1. 파이썬 설치
- https://www.python.org/downloads/ 에서 최신 버전 설치
- 설치 화면에서 **"Add Python to PATH"** 체크 꼭 하기

### 1-2. 파일 준비
이 폴더(app.py, data_sources.py, forecast.py, requirements.txt)를
한 폴더에 모아둡니다.

### 1-3. 필요한 프로그램 설치

**먼저, "터미널"이 뭔지부터.**
터미널(윈도우에서는 "명령 프롬프트")은 마우스 대신 **글자를 입력해서 컴퓨터에
명령을 내리는 검은 창**입니다. 딱 두 번(설치 1번, 실행 1번)만 쓰면 되니
겁먹지 않으셔도 됩니다.

**① 파일이 있는 폴더에서 터미널 열기**

- **윈도우**: 파일 5개를 모아둔 폴더를 연다 → 폴더 안의 **빈 공간을
  Shift 키를 누른 채 마우스 오른쪽 클릭** → "여기에 PowerShell 창 열기"
  또는 "터미널에서 열기" 클릭. (이 메뉴가 없으면: 폴더 위쪽 주소창을
  클릭해서 `cmd` 라고 입력하고 Enter)
- **맥**: 파일 5개를 모아둔 폴더를 Finder에서 찾는다 → 폴더를
  **마우스 오른쪽 클릭** → "폴더에서 새로운 터미널 열기" 클릭.
  (이 메뉴가 없으면: 시스템 설정 → 키보드 → 키보드 단축키 → 서비스에서 켤 수 있음)

검은(또는 흰) 창이 뜨면 성공입니다.

**② 아래 한 줄을 복사해서 창에 붙여넣고 Enter**

```bash
pip install -r requirements.txt
```

- 이 명령의 뜻: "requirements.txt에 적힌 부품(패키지)들을 자동으로 내려받아
  설치해줘"입니다. 앱을 만들 때 쓴 부품들을 사장님 컴퓨터에도 똑같이 깔아주는 과정이에요.
- 글자가 주르륵 올라가면서 1~3분 정도 걸립니다. **마지막에 에러 없이 멈추면 끝.**
- `pip is not recognized` 같은 빨간 글씨가 나오면: 파이썬 설치 때
  "Add Python to PATH"를 체크 안 한 경우입니다. 파이썬을 지우고
  다시 설치하면서 그 체크박스를 켜주세요.

**③ (선택) 예측 모델까지 쓰고 싶다면 한 줄 더**

```bash
pip install prophet
```

- Prophet은 설날·추석 같은 연휴 효과까지 계산하는 통계 예측 부품입니다.
- **설치가 안 되거나 에러가 나면 그냥 무시하고 넘어가세요.** 앱이 알아서
  더 단순한 계절성 그래프 방식으로 작동하고, 화면에 노란 안내문으로 알려줍니다.
  나중에 다시 설치하면 자동으로 예측 모델로 바뀝니다.

## 2. 실행

방금 그 터미널 창에(닫았다면 1-3 ①처럼 다시 열고) 아래 한 줄을 붙여넣고 Enter:

```bash
streamlit run app.py
```

- 이 명령의 뜻: "app.py라는 앱을 켜줘"입니다. 앞으로 앱을 켤 때마다
  이 한 줄만 쓰면 됩니다.
- **`streamlit ... 인식되지 않습니다` 오류가 뜨면**: 대신
  `python -m streamlit run app.py` 를 쓰세요. (자세한 건 5번 FAQ 첫 항목)
- 몇 초 뒤 **브라우저가 저절로 열리면서 대시보드가 나타납니다.**
- 안 열리면: 터미널에 `Local URL: http://localhost:8501` 이라는 글자가
  보일 거예요. 그 주소를 복사해 크롬 주소창에 붙여넣으세요.
  (localhost = "내 컴퓨터 안에서만 열리는 주소"라는 뜻이라,
  다른 사람은 못 보니 안심하셔도 됩니다. 남에게 공유하려면 4번 배포 참고)

**끄는 법**: 터미널 창에서 `Ctrl + C`를 누르거나 창을 닫으면 앱이 꺼집니다.

이 상태에서 **이미 샘플 데이터로 모든 기능이 작동**합니다. 키 발급 전에
화면 구성부터 편하게 둘러보세요.

## 3. 실데이터 연결 (선택)

### 3-1. 한국관광공사 키 (수요 예측 · 연령대)
1. https://www.data.go.kr 회원가입
2. "한국관광공사_빅데이터_지역별 방문자수" 검색 → **활용신청**
3. 승인 후 발급된 **일반 인증키(서비스키)** 복사
4. 앱 사이드바 → "🔑 API 키 입력" → 붙여넣기

### 3-2. 네이버 키 (예능 여행지)
1. https://developers.naver.com/apps/#/register 접속
2. "검색" API 추가 → 애플리케이션 등록
3. **Client ID / Client Secret** 복사
4. 앱 사이드바에 붙여넣기

> **배지 안내:** 키를 넣어도 아직은 🟡 샘플로 표시됩니다. 실데이터 연결 함수
> (아래 3-3)가 완성돼야 🟢 실데이터로 바뀝니다. 이는 샘플 수치를 실데이터로
> 착각해 광고 의사결정에 쓰는 일을 막기 위한 안전장치입니다.

### 3-3. 실데이터 함수 채우기 (개발 도움 필요)
`data_sources.py` 안의 두 함수만 채우면 됩니다:
- `_fetch_datalab_visitors()` — 지역별 월 방문자(내국인) 파싱
- `_fetch_datalab_age()` — 연령대별 비중 파싱

완성 후 같은 파일 상단의 `LIVE_IMPLEMENTED = False`를 `True`로 바꾸면
사이드바 배지가 🟢 실데이터로 표시됩니다.

함수 안에 어떤 API를 어떻게 호출할지 주석으로 적어두었습니다.
이 부분은 데이터랩의 실제 지역코드 매핑이 필요해서, 키를 발급받은 뒤
저(Claude)에게 "키 발급했어, 이 함수 채워줘"라고 하시면 완성해 드립니다.

처음 보여주신 railway 사이트처럼 인터넷 주소로 공유하려면:

**가장 쉬운 방법 — Streamlit Community Cloud (무료)**
1. https://github.com 가입 후 이 파일들을 저장소(repository)에 업로드
2. https://share.streamlit.io 접속 → GitHub 연결
3. 저장소 선택 → `app.py` 지정 → Deploy
4. 몇 분 뒤 `https://여러분-앱.streamlit.app` 주소 생성

이 과정도 막히면 단계별로 도와드릴게요.

## 5. 자주 묻는 질문

**Q. `streamlit ... 인식되지 않습니다` 오류가 떠요. (윈도우 최다 빈출)**
"streamlit 명령어를 컴퓨터가 못 찾겠다"는 뜻입니다. 원인이 뭐든 아래 방법으로
해결됩니다. 터미널에 이 한 줄을 입력하세요:
```bash
python -m streamlit run app.py
```
- 대시보드가 열리면 성공. **앞으로 앱을 켤 때 항상 이 명령을 쓰세요.**
- `No module named streamlit` 이 나오면 설치가 안 된 것이니
  `python -m pip install -r requirements.txt` 로 설치 후 다시 실행.
- `python` 자체가 인식 안 되면 `python` 대신 `py` 로 바꿔 시도.
  (예: `py -m streamlit run app.py`)

**Q. 키가 없으면 가짜 데이터인가요?**
네. 화면 구조와 기능을 미리 확인하기 위한 샘플입니다. 키를 넣고 실데이터 함수를
채우면 진짜 데이터로 바뀝니다.

**Q. 예측이 '계절성 그래프'로 나와요.**
데이터가 24개월 미만이거나 Prophet 미설치 상태입니다. 데이터가 쌓이고 Prophet을
설치하면 자동으로 통계 예측 모델로 전환됩니다.

**Q. 예능 여행지가 실제 방영 정보와 달라요.**
네이버 검색을 자동 수집한 결과라 정확도에 한계가 있습니다. 중요한 항목은
`data_sources.py`의 `_sample_spots()`처럼 직접 입력해 보정할 수 있습니다.
카드 라벨의 프로그램명도 자동 감지라서, 자주 다루는 프로그램은
`KNOWN_SHOWS` 목록에 추가해 두면 인식률이 올라갑니다.

**Q. 예측 수치를 보고서에 쓰고 싶어요.**
수요 예측 탭 하단의 "📋 예측 수치 표로 보기 / 내려받기"를 열면
CSV 내려받기 버튼이 있습니다. 엑셀에서 바로 열립니다.

**Q. requirements.txt에 새로 추가된 건 없나요?**
없습니다. v2 개선은 전부 기존 패키지와 파이썬 내장 기능으로 구현돼,
설치 목록은 처음 그대로입니다.
